# 🔬 Система извлечения фармакокинетических данных

Автоматическое извлечение фармакокинетических параметров из баз данных (PubMed, DrugBank, GRLS) и рекомендация дизайна исследования биоэквивалентности с использованием LLM.

## 📋 Возможности

- ✅ Извлечение данных из PubMed/PubMed Central
- ✅ Поддержка DrugBank, Simcyp, GRLS (в разработке)
- ✅ Извлечение параметров: Cmax, Tmax, AUC, T½, Clearance, Vd, CVintra
- ✅ **Рекомендация дизайна** исследования биоэквивалентности через LLM
- ✅ **Автоматический расчёт размера выборки** по стандартным формулам FDA/EMA
- ✅ **Экспорт результатов в JSON** для интеграции с другими системами
- ✅ Поддержка **open-source моделей** (Ollama, LM Studio)
- ✅ Легкая смена LLM провайдера

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install requests
```

### 2. Запуск с MockProvider (тестовый режим)

```bash
python main.py --drug "Amlodipine" --max-results 5
```

### 3. Настройка реальной LLM модели

Отредактируйте `llm_config.py` и раскомментируйте нужный провайдер.

## 🤖 Настройка LLM провайдеров

### Вариант 1: Ollama (Рекомендуется для начала) 🌟

**Ollama** - самый простой способ запустить open-source модели локально!

#### Установка Ollama:

```bash
# Linux/Mac
curl -fsSL https://ollama.ai/install.sh | sh

# Windows
# Скачайте с https://ollama.ai
```

#### Загрузка модели:

```bash
# Выберите одну из моделей
ollama pull llama3        # Meta Llama 3 (рекомендуется)
ollama pull mistral       # Mistral 7B (быстрая)
ollama pull mixtral       # Mixtral 8x7B (мощная)
ollama pull qwen          # Qwen (хорошо работает с медицинскими данными)
```

#### Запуск Ollama сервера:

```bash
ollama serve
```

#### Настройка в llm_config.py:

```python
def get_llm_provider():
    return OllamaProvider(
        model="llama3",  # или "mistral", "mixtral", "qwen"
        base_url="http://localhost:11434"
    )
```

#### Запуск анализа:

```bash
python main.py --drug "Amlodipine" --max-results 10
```

## 📝 Формат PKRecord

```python
@dataclass
class PKRecord:
    source: str              # Источник данных (PubMed, DrugBank)
    drug: str                # Название препарата

    # Параметры препарата
    dosage_form: Optional[str]
    dosage: Optional[float]

    # Фармакокинетические параметры
    cmax: Optional[float]
    tmax: Optional[float]
    auc: Optional[float]
    t_half: Optional[float]
    clearance: Optional[float]
    volume_distribution: Optional[float]
    cv_intra: Optional[float]      # ⭐ Ключевой параметр!

    # Метаданные
    title: Optional[str]
    study_id: Optional[str]
    n_subjects: Optional[int]
```

## 🎯 Критерии выбора дизайна

| Дизайн | CVintra | T½ | N участников |
|--------|---------|-----|--------------|
| 2×2 Cross-over | ≤ 30% | < 24h | 24-28 |
| 3-way Replicate | 30-50% | < 24h | 36-42 |
| 4-way Replicate (RSABE) | > 50% | < 24h | 48-60 |
| Параллельный | N/A | > 24h | 150+ |

## 🧮 Расчёт размера выборки

Система автоматически рассчитывает размер выборки на основе:

### Формулы для различных дизайнов:

**2×2 Cross-over:**
```
N = 2(Z_α/2 + Z_β)² × σ² / (ln Θ₁)²
```

**3-way Replicate:**
```
N = 3(Z_α/2 + Z_β)² × σ² / (ln Θ₁)²
```

**4-way Replicate (RSABE):**
```
N = 2(Z_α/2 + Z_β)² × σ² / (ln Θ₁)²
```

**Параллельный:**
```
N = 2(Z_α/2 + Z_β)² × σ² / δ²
```

где:
- **σ² = (ln(1 + CV))²** - внутрисубъектная дисперсия
- **Z_α/2 = 1.96** - критическое значение для α = 0.05 (95% ДИ)
- **Z_β = 0.84** - критическое значение для power = 0.80 (80% мощность)
- **Θ₁ = 0.80** - нижняя граница биоэквивалентности
- **Θ₂ = 1.25** - верхняя граница биоэквивалентности
- **δ = ln(Θ₂) - ln(Θ₁)** - для параллельного дизайна

### Учёт выбывания (dropout):

```
N_итоговое = N_базовое / (1 - dropout_rate)
```

По умолчанию: **dropout_rate = 20%** (можно настроить)

### Пример расчёта:

Для препарата с CVintra = 25%:
1. σ² = (ln(1 + 0.25))² = (ln(1.25))² = (0.223)² = **0.0497**
2. N = 2 × (1.96 + 0.84)² × 0.0497 / (-0.223)² = **15.7** ≈ **16 участников**
3. С учётом 20% dropout: 16 / 0.80 = **20 участников**

## 📄 Формат выходных данных

Результаты сохраняются в двух форматах:

### 1. Текстовый файл (`.txt`)
Подробное описание с обоснованием и расчётами

### 2. JSON файл (`.json`)
Структурированные данные для интеграции:

  "t_half_used": 8.5,

    "reasoning": "Детальное обоснование расчёта..."
